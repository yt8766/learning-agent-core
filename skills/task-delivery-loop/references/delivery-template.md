# Delivery Template

可按下面结构整理最终交付，保持简洁但信息完整。

## 1. 需求与范围

- 本次要解决什么问题
- 影响哪些模块 / contract / 页面 / 流程

## 2. 核心改动

- 改了哪些关键逻辑
- 为什么这样改
- 是否有结构性重构或拆分

## 3. 验证结果

- 实际执行了哪些命令
- 哪些通过
- 哪些因 blocker 未执行
- blocker 是否属于本轮改动

## 4. 文档同步

- 更新了哪些文档 / 技能 / 注释
- 是否清理了过时说明

## 5. 清理结果

- 删除了哪些无用文件 / 旧规范 / 旧导出 / 旧实现
- 哪些内容因为兼容或迁移原因暂时保留

## 6. 风险与关注点

- 仍有哪些潜在风险
- reviewer 应重点看哪些链路

## 7. Commit / PR 建议

### Commit Message

```text
<type>: <summary>
```

### PR Description

```md

- ...

## Testing

- ...

## Risks

- ...
```
